# 微信文章噪声与排版参考

`fetch_clean.mjs` 已经移除了**固定 UI chrome**（滑动栏、扫一扫/小程序、底部操作栏、
重复的账号名抬头、小说阅读器广告）。本文件描述需要**语义判断**才能处理的部分：
嵌入正文中的软文/导流，以及排版结构的还原。

---

## 一、需要清除的嵌入式软文（按需删除，需判断）

这些内容混在正文里，措辞每次都不同，无法用脚本可靠匹配。判断标准：**它是不是文章
本身的信息？** 如果只是引导读者做动作（关注、回复、点赞、加群、买课），就删掉。

常见类型与特征关键词：

| 类型 | 特征关键词 |
|------|-----------|
| 公众号导流 | 关注我们、点击上方蓝字、星标公众号、设为星标、第一时间收到 |
| 后台互动 | 后台回复【xxx】、回复关键词、私信领取 |
| 粉丝福利/抽奖 | 粉丝福利、点赞最高者送、抽奖、免费领取、扫码进群 |
| 引导互动话术 | 觉得有用点个赞、点赞+在看、转发给需要的人、评论区聊聊（纯引导无信息时） |
| 二维码/名片 | 长按识别二维码、扫码关注、添加微信、商务合作请联系 |
| 课程/带货广告 | 限时优惠、原价xxx现价、立即购买、报名链接 |
| 互推/荐读 | 推荐阅读、往期精彩、点击下方卡片 |

判断要领：
- **保留**与文章主题相关的实质信息，即使带 emoji（如 `📅 2026.06.03`、`💥 恐怖性能：…`
  这类是正文小标题/要点，要留）。
- **删除**纯粹引导用户做动作、对内容理解无贡献的段落。
- **数据来源 / 参考资料 / 免责声明**通常保留——它们是文章的一部分。
- 拿不准时，倾向保留信息、只删明显的营销话术，避免误删正文。

---

## 二、排版结构还原

微信编辑器没有真正的标题层级，作者用 **emoji 前缀 + 加粗 + 分隔线** 模拟标题。
Defuddle 输出后这些都变成了普通段落。任务是**根据原文的逻辑结构**还原 Markdown 标题层级。

### 识别信号

- **一级标题**：文章主标题（frontmatter 已有 `title`，正文里重复出现的大标题可删或转为 `#`）。
- **章节标题**：常带序号/强调 emoji，如 `🎯 第一炸：…`、`💣 第四炸：…`、`🔮 结语：…`、
  `💰 时间线：…`。→ 转为 `##`。
- **小节标题**：章节下的副标题或日期戳，如 `📅 2026.06.03 · …`、`📉 昨日·安全神话`。
  → 转为 `###`，或作为该节的引导句保留，视层级而定。
- **要点强调**：整段加粗的短句（`**💥 恐怖性能：**…`）是正文要点，**不要**升成标题，保留为段落或列表。

### 处理原则

1. **忠于原文逻辑**：标题层级要反映作者本来的结构，不要自创或重排章节顺序。
2. **层级一致**：同一类章节用同一级标题（所有"第 N 炸"都用 `##`）。
3. **正文不动**：只调整标题层级、清理软文、合并被错误拆断的段落；**不改写、不增删正文内容**。
4. **列表归并**：开头那种被拆成 `- 1.` / 换行 / 正文的导读列表，合并成正常的有序列表项。
5. **图片保留**：正文里的 `![](https://mmbiz.qpic.cn/...)` 是文章配图，保留。
6. **清理残留**：删除孤立的 emoji 行、重复出现的标题行、连续空行。

### 示例

原始（Defuddle 输出）：
```
🎯 第一炸：红队至暗时刻——内鬼倒卖天价模型

📅 2026.06.03 · 红队测试变成黑色交易

5万Token搞定macOS，一秒52 Token狂飙

事情源于Anthropic的Claude开发者后台中...
```

还原后：
```
## 🎯 第一炸：红队至暗时刻——内鬼倒卖天价模型

> 📅 2026.06.03 · 红队测试变成黑色交易 · 5万Token搞定macOS，一秒52 Token狂飙

事情源于Anthropic的Claude开发者后台中...
```
（日期戳 + 导语合并为引用块作为小节提要；正文原样保留。）
